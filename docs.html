Welcome to SolScript

This is the base documentation landing page for SolScript. It is designed to feel like a proper documentation product: calm, readable, and structured enough to scale as the language grows.

Use the sidebar to move between the major sections, or start with the quick-start cards below to get oriented fast.

Quick start

Install the toolchain

Add the compiler, wire up your editor, and make sure the first build path is working before you dig into deeper language features.

Run a hello world

Confirm that the environment is healthy with the smallest possible program.

Read the reference

Use the language reference when you need exact syntax, behavior, or examples.

Core ideas

1

Readable structure

Syntax should stay approachable so that project files remain easy to navigate over time.

2

Compiled output

SolScript is positioned as a compiled systems language rather than an interpreted scripting layer.

3

Room to grow

The documentation structure should scale as features, modules, and libraries are added.

Syntax overview

SolScript aims to keep first-pass code short and direct. This keeps examples readable and makes docs pages easier to scan. 

// A minimal program
print("Hello, World!");
