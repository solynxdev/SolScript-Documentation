Built like a language people want to keep using.

SolScript aims for the familiar clarity of a scripting language with the performance profile of a compiled systems toolchain.

01

Performance

Native compiled output with room for optimization, low-level control, and serious application work.

02

Simplicity

Fewer concepts in the way means less friction when you are moving from an idea to working code.

03

Consistency

A language surface that stays predictable across modules, features, and large projects.

What it looks like

A language can be approachable without feeling shallow. The first line should make sense, and the hundredth should still feel tidy.

Hello, World.

SolScript keeps the hello-world moment short and direct. 

print("Hello, World!");

Where it fits

The language is meant to scale from small tools to larger systems work without making the project feel heavier than it needs to be.

Command-line tools

Build fast utilities with a small surface area and straightforward control flow.

Game and engine code

Keep latency-sensitive code close to the metal while still reading cleanly in larger systems.

Reusable libraries

Structure code for long-term maintenance, not just for the first pass.
